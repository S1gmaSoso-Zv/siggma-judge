# SIGGMA JUDGE

SIGGMA JUDGE — это локальная платформа для практики в алгоритмическом программировании.

В отличие от классических тестирующих систем (вроде LeetCode или Яндекс.Контеста), которые просто прогоняют код через набор тестов, этот проект сфокусирован на код-ревью. Интегрированная локальная нейросеть выступает в роли ментора: она анализирует написанный код, оценивает асимптотическую сложность алгоритма (Big O), указывает на неоптимальные участки (например, вложенные циклы) и дает подсказки.

Всё работает полностью локально на вашем железе — без привязки к интернету, платным API и сторонним серверам.

**Важное замечание:** В основе проверки лежит локальная генеративная модель (по умолчанию Mistral). Как и любая LLM, она может иногда ошибаться, галлюцинировать, не замечать очевидных багов или, наоборот, придираться к мелочам. Относитесь к её комментариям как к советам напарника по парному программированию, а не как к абсолютной истине.

## Основной функционал

* **Генерация задач:** Система умеет "на лету" придумывать уникальные алгоритмические задачи разного уровня сложности — от базовых до олимпиадного уровня.
* **Изолированное выполнение (Sandbox):** Написанный код не просто читается нейросетью, а физически запускается на сервере в безопасной среде. Поддерживается перехват ошибок, обработка бесконечных циклов (timeout) и передача пользовательского ввода (stdin).
* **«No-Spoiler» Ревью:** Системный промпт ИИ настроен достаточно жестко. Нейросети запрещено писать готовый код решения — она должна лишь указывать на логические или синтаксические ошибки, заставляя пользователя думать самостоятельно.
* **Веб-редактор:** Минималистичный и легкий интерфейс на базе CodeMirror с поддержкой синтаксиса Python, нумерацией строк и темной темой.

## Стек технологий

* **Backend:** Python 3, FastAPI, uvicorn. Для изоляции выполнения кода используется встроенный модуль `subprocess`.
* **Frontend:** Чистый HTML, CSS и Vanilla JS + библиотека CodeMirror. Никаких перегруженных фреймворков, всё работает в одном браузере.
* **AI-движок:** Ollama с моделью Mistral. Общение с LLM реализовано через асинхронный HTTP-клиент `httpx`.

## Инструкция по локальному запуску

Для работы потребуется установленный Python (3.9+) и сервис Ollama.

**1. Подготовка ИИ**
Установите [Ollama](https://ollama.com/) и загрузите легковесную модель Mistral, выполнив в терминале команду:

```bash
ollama run mistral

```

Убедитесь, что приложение Ollama запущено и работает в фоне.

2. Настройка проекта
Склонируйте этот репозиторий на свой компьютер:

```Bash
git clone [https://github.com/S1gmaSoso-Zv/siggma-judge.git](https://github.com/S1gmaSoso-Zv/siggma-judge.git)
```
Затем перейдите в папку проекта:

```Bash
cd siggma-judge
```
3. Виртуальное окружение
Создайте изолированное окружение, чтобы не засорять систему:

```Bash
python -m venv venv
Активируйте его.
```
Для Windows:

```Bash
.\venv\Scripts\activate
```
Для Linux/macOS:

```Bash
source venv/bin/activate
```
4. Зависимости
Установите необходимые пакеты для работы сервера:

```Bash
pip install fastapi uvicorn httpx pydantic
```
5. Запуск
Запустите бэкенд:

```Bash
uvicorn backend.main:app --reload
```
После того как в консоли появится сообщение об успешном старте, просто откройте файл frontend/index.html в любом браузере.
